# GitHub AI レビュー 指示書

GitHub 連携における AI コードレビューおよび生成の基準を定義する。

## 言語ポリシー

- 日本語で記述する。
- 過去形ではなく命令形（imperative mood）で記述する。
- ですます調は使わず、常体（〜する。〜確認する。）で統一する。
- 同じ意味の用語は表記ゆれが出ないように統一する。
- 絵文字や強調記号（\*や\_）は使わない。

## コードルール

以下の原則に従いコードレビューを実施する。

- DRY（Don't Repeat Yourself）
- SRP（Single Responsibility Principle）
- KISS（Keep It Simple, Stupid）
- YAGNI（You Aren't Gonna Need It）
- OAOO（Once And Only Once）
- 整合性チェック（Consistency Check）

これらの原則に違反する箇所を検出し、改善案を提示する。

## 禁則事項

以下の行為を禁止する。

- 過剰な try-catch を行わない
- a11y 対応およびレスポンシブ調整を行わない

## 要件

以下の要件を満たすこと。

### 1. 効率化

- Astro の機能を効率的に使用する。
- microCMS の機能を効率的に使用する。
- 慣例およびデファクトスタンダードに従う。

### 2. 保守性

- 可読性を優先する。
- ファイル構造を整理する。
- 論理的な並べ替えを行う。
- 不要な複雑性を排除する。

### 3. PR 内容まとめテンプレート

```markdown
## 変更内容

| ファイル・フォルダ | 変更内容     |
| ------------------ | ------------ |
| `{path/to/file}`   | {変更の概要} |
```
